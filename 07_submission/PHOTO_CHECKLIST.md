# Photo & Video Checklist for Hackster Submission

## Required Photos
- [ ] **Hero shot**: Board + microphone on a desk, well-lit, top-down view
- [ ] **Wiring closeup**: INMP441 to UNO Q connections clearly visible
- [ ] **Serial Monitor**: Arduino IDE showing inference output (all 4 classes)
- [ ] **Dashboard – idle**: Browser showing green badge
- [ ] **Dashboard – presence**: Browser showing yellow badge
- [ ] **Dashboard – anomaly**: Browser showing red badge
- [ ] **Dashboard – manual_trigger**: Browser showing blue badge
- [ ] **Dashboard – history**: Event history table with multiple entries

## Required Video
- [ ] **Demo video (30–60 s)**: Shows idle → presence → anomaly → manual_trigger in sequence
  - Tip: use screen recording for the browser + phone camera for the board simultaneously
  - Upload to YouTube (unlisted OK) and embed in Hackster article

## Photo Tips
- Use natural light or softbox; avoid harsh shadows
- Clean desk background
- Show the board label "Arduino UNO Q" clearly
- Show the browser URL bar with the local IP to demonstrate local-only operation

## Hackster Submission Checklist
- [ ] Article draft complete (HACKSTER_ARTICLE.md)
- [ ] All photos taken and uploaded
- [ ] Demo video uploaded and linked
- [ ] GitHub repository made public
- [ ] BOM filled in
- [ ] Code repository linked in article
- [ ] Submitted before contest deadline
